# 将Primetime身份验证服务器端数据集成到Adobe Analytics

>[!NOTE]
>
>此页面上的内容仅供参考。 使用此API需要获得Adobe的当前许可证。 不允许未经授权使用。

Adobe Primetime身份验证的客户希望在Adobe Analytics功能板中查看Primetime身份验证(Adobe Pass)服务器端数据，以便更轻松地使用。

数据将用于跟踪重要的TVE量度，例如每个MVPD的验证转化率、基于MVPD用户ID的独特用户等。

如果客户端实施已存在，则不会替换该实施，因为在没有访客ID的情况下，无法对用户活动进行跟踪，超出下面的特定事件。 如果客户在传递调用中提供访客ID，则我们可以解锁另一种类型的Analytics集成（实时），该集成可将所有传递事件与现有客户数据相结合，有关此新类型可能集成的更多详细信息，请参阅此处：&quot;[在Primetime身份验证中使用Experience CloudID](/help/authentication/exp-cloud-id-authn.md)&quot;

## 包含的量度 {#metrics-included-int-authn-analyt}

| 事件 | 描述 |
|----------------------------|----------------------------------------------------------------------------------------------------------------------|
| 请求了AuthN | 启动的身份验证流数 |
| AuthN挂起 | 成功生成的身份验证令牌数量（无论客户端是否实际获取了该令牌） |
| AuthN OK | 用户成功获取的身份验证令牌数量 |
| 请求了AuthZ | 尝试的授权数 |
| AuthZ OK | 成功授权的数量 |
| AuthZ失败 | MVPD在应用程序级别拒绝的授权数 |
| 播放请求 | 生成的短媒体令牌数（与播放请求数同化） |
| 请求注销 | 启动的注销流数 |
| 注销已完成 | 成功完成的注销流数 |
| 注销失败 | 失败注销流的数量 |
| 请求的预授权 | 启动的预授权流数 |
| 预授权确定 | 预授权资源的成功预授权事件数 |
| 拒绝预授权 | 拒绝预授权的资源的预授权事件数 |
| 预授权失败 | 失败的预授权事件数 |

| Adobe Analytics名称 | 描述 |
|------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 渠道 | 用于执行授权请求的请求者ID |
| MVPD | 负责向用户授予权限的MVPD |
| 代理 | 代理MVPD（将“直接”用于直接集成） |
| SDK类型 | 使用的客户端SDK(Flash、HTML5、Android本机、iOS、无客户端等) |
| SDK版本 | Adobe Primetime身份验证客户端SDK的版本 |
| 资源ID | 授权请求中涉及的实际资源标题（如果提供，则从MRSS有效负载中提取为项目/标题） |
| AuthZ错误类型 | 失败的原因(由Adobe Primetime身份验证报告) <br/> 以下是最常用的值 <br/> **noAuthZ** = MVPD答复用户的包中没有渠道<br/> **网络** =我们无法联系MVPD（MVPD在呼叫时遇到问题，且未回复）<br/> **norefreshtoken** =这完全适用于OAuth实施，如果用户更改其密码或MVPD因某些原因拒绝该密码，则可能会导致此操作。 通常会导致新的身份验证<br/> **不匹配** =如果请求来自与具有身份验证令牌的设备不同的设备，则为。 如果用户尝试欺骗系统，但大多数情况发生在旧JavaScript SDK的上下文中，在该上下文中，设备ID使用IP地址作为计算的一部分，则会生成结果。 如果用户在家观看TVE，然后在工作时触发该错误，并且必须再次进行身份验证<br/> **无效** =无效请求，缺少或无效参数<br/>  **authzNone** =程序员能够拒绝特定渠道xMVPD组合的授权。 这由程序员有权访问的后端API触发<br/> **欺诈** 就是保护机制。 如果用户授权失败，然后在短时间间隔（秒）内再次请求它多次，则我们会直接拒绝调用。 当程序员在实施中遇到错误，如果失败，程序员会不断要求授权时，通常会发生这种情况。 |
| 令牌类型 | 当由于“全部验证”和“全部验证”而创建令牌时，我们必须知道降级措施导致的变化。<br/> 它们是：<br/> &quot;normal&quot; =正常大小写<br/> &quot;authnall&quot; =启用AuthN All时<br/> &quot;authzall&quot; =启用AuthZ时全部<br/>  &quot;hba&quot; =启用HBA时 |
| 无客户端设备类型 | 当前用于无客户端的设备平台（替代）。<br/> 值可以是：<br/> 不适用 — 事件并非源自无客户端SDK<br/> 未知 — 由于 **无客户端API** 为可选，则有一些调用不包含任何值。<br/> 通过 **无客户端API**. 例如xbox、appletv和roku。 |
| MVPD用户ID | 替换基于Cookie的访客ID |


## 详细信息 {#details-int-authn-analyt}

* 将通过Adobe Analytics数据插入API，在特定报表包中逐个事件插入量度
* 插入内容将被批处理，每30分钟发送一次。 因此，报表需要加盖时间戳
* 每个客户将有一个或多个报表包。 一个请求者ID（渠道）将仅映射到一个报表包。 多个请求者ID只能映射到一个报表包。
* 可以提供历史数据，但由于流量/性能方面的考虑，需要特别注意。
* 独特访客变量设置为MVPD用户ID
* 无法配置事件和eVar的映射。
