# PrimetimeAd Insertion20.12.1发行说明

PrimetimeAd Insertion20.12.1发行说明介绍了2020年PrimetimeAd Insertion的新增或更改内容、已解决的问题和已知问题。

## PTAI 20.12.1的新增功能

**时间：** 2020年12月8日（星期二）东部时间半夜1点至4点

**更改**

* 包括修补程序，用于解决2020年11月30日遇到的PrimetimeAd Insertion中的间歇性客户端连接(5xx)问题。

### 版本20.9.3

**时间：** 2020年9月30日，星期三，东部时间凌晨3点30分至6点30分

**更改**

* 添加了BootstrapAPI参数 `ptparallelstream`. 这允许其播放器请求并行进行CMAF深层链接音频或视频流的客户确保音频和视频曲目中的广告一致。 将参数值设置为true可启用此功能，或忽略以禁用。

### 版本20.9.2

**时间：** 2020年9月15日，星期二，东部时间凌晨3:30至早上6:30

**增强功能**

* 支持使用包含非线性广告类型 `EXT-X-MARKER` 标记之间。
有关详细信息或要启用此功能，请与技术支持代表联系。

* 如果提供商响应时间过长，则支持限制整体广告解决时间。 要启用限制，请设置引导API参数 `ptadtimeout` 转换为以毫秒为单位的值。

   >[!NOTE]
   >
   >此超时仅适用于广告请求，而不适用于广告创意请求。

### 版本20.9.1

**时间：** 2020年9月1日，星期二东部时间凌晨3:30至早上7:30

**更改**

* 修复了使用HLS/CMAF的客户遇到的问题：EXT-X-MAP有时缺少CDN令牌，或者EXT-X-MAP标记有时从DVR窗口错误地转出。

**增强功能和修复**

维护更新。

### 版本20.7.1

**时间：** 2020年7月9日，星期四，东部时间凌晨3点至5点

**新增功能和增强功能**

* SCTE35增强功能，可使用“提供程序广告开始/结束”消息或“中断开始/结束”消息来标识提示。

* 更新了X-ADOBE-AI-X1标头，提供了其他信息以帮助进行故障排除。

* 增强的量度聚合。

* 会话统计面板的增强SSAI控制台仪表板

### 版本20.6.2

**时间：** 2020年6月18日，星期四，东部时间凌晨3点至4点

**增强功能**

改进了需要毫秒精度的视频客户端的流同步。 联系Adobe支持以启用毫秒精度 `#EXT-X-PROGRAM-DATE-TIME tags`.

### 版本20.6.1

**时间：** 2020年6月2日，星期二东部时间凌晨03点至05点

**新增功能**

请联系Adobe支持，以通过服务器端配置启用以下新功能：

* 清单操作：现在可以在HTTP和HTTPS之间转换HLS区段和资源URL，通过减少后端请求上的TLS握手来提高性能。 它也可用于将广告/内容片段统一到相同的CDN上。

* 长格式VOD：改进了API，以使用长格式VOD资产保持会话正常运行。

**错误修复**

* 修复了始终根据http协议请求WebVTT片段的问题，而不管请求的原始协议是什么。

* 修复了从广告切换回内容时，从播放列表顶部删除EXT-X-DISCONTINUITY标记的问题。 请联系Adobe支持以启用此修复。

### 版本20.5.1

**时间：** 2020年5月5日，星期二东部时间凌晨04点至05点

* 修复了一个问题，以确保在发送If-Modified-Since标头时提供正确的CORS标头。

* CRS功能板上的错误修复。

* 维护更新。

### 版本20.3.4

**时间：** 2020年4月1日，星期三，东部时间凌晨3点至4点

* 修复了在VOD/WebVTT中插入广告后导致字幕不同步的问题。

* 安全更新。

### 版本20.3.3

**时间：** 2020年3月26日，星期四，东部时间凌晨3点至4点

* SSAI 4XX和5XX响应现在可以正确提供与CORS相关的标头，从而允许跨域javascript webview客户端成功读取错误响应。

* 修复了X-Forwarded-For标头的问题，该问题导致在将IPv6地址传递到广告服务器时，其URL编码不正确。

* 修复了CMAF/去模糊音频流的问题，在某些情况下，EXT-X-MEDIA-SEQUENCE号的递增将不正确。

### 版本20.3.2

**时间：** 2020年3月11日，星期三，东部时间凌晨5:30至早上7:00

* 改进了SCTE35信号处理。

* 维护更新。

### 版本20.3.1

**时间：** 2020年3月5日，星期四，东部时间凌晨02:30至凌晨04:30

* 性能改进：

   * 添加了对主控/媒体m3u8清单的缓存支持。 这些清单现在响应Cache-Control： public和Max-Age标头，这通常可以提高视频启动性能。

   * 增加了对强制通过http获取https创意的支持，这还可以提高视频启动性能。

* 安全和维护修复。

### 版本20.2.1

**时间：** 2020年2月13日，星期四东部时间凌晨04点30分至05点30分

* 添加了对拼接广告资产的支持，这些广告资产包含基于语言/编解码器/比特率的多个纯音频流。
* 次要的性能增强和维护更新。

### 版本20.1.3

**时间：** 2020年1月28日，星期二，东部时间凌晨2:00至3:00

* **VMAP支持nbc CueFormat FER**

   在以下情况下，将提示从FER流转换为FW时间线覆盖参数 `ptcueformat=nbc` 使用，并且流是具有清单内提示和内置广告的VOD流。

* 在转发到第三方广告提供商/CDN之前，清理HTTP标头中的用户代理字段。

* 在发送到Auditude和其他广告提供商CDN之前，从用户代理HTTP标头中过滤掉控制/不可打印字符（ASCII代码&lt; 32）。 此类无效标头的Auditude Ad-Call过去会失败。

* 从NetStorage组中清除旧V1对象，以使对象计数在Akamai的安全限制内。

### 版本20.1.2（修补程序）

**时间：** 2020年1月20日，星期一东部时间凌晨02:00至凌晨03:00

* 维护更新。

### 版本20.1.1
